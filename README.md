# Psychological Test

Веб-приложение для прохождения психологических тестов на FastAPI.

Поддерживает тесты с несколькими шкалами, расчет результатов по правилам,
сохранение истории прохождений и импорт структуры теста из внешних файлов.

## Запуск

```bash
python -m venv .venv
source .venv/Scripts/activate
pip install -r requirements.txt
python seed.py
uvicorn main:app --reload
```

Адрес приложения:

```text
http://127.0.0.1:8000
```

## Основные файлы

`main.py`  
Маршруты FastAPI, прохождение теста, прием ответов, расчет итогов и отображение
страниц.

`models.py`  
SQLAlchemy-модели базы данных: тесты, шкалы, вопросы, варианты ответов, правила
начисления баллов, сессии прохождения и результаты.

`database.py`  
Подключение к SQLite, создание `engine`, `SessionLocal` и базового класса
моделей.

`seed.py`  
Создание демонстрационных тестов и заполнение базы `app.db`.

`test_importer.py`  
Общий импорт теста в базу данных из единой структуры данных. Используется для
демо-данных и внешних парсеров.

`import_excel.py`  
CLI-команда для импорта теста из Excel-файла.

`parsers/excel.py`  
Парсер Excel-файла. Преобразует листы Excel в структуру, которую принимает
`test_importer.py`.

`templates/`  
HTML-шаблоны страниц приложения.

`static/`  
CSS-стили.

`app.db`  
SQLite-база данных приложения.

## Структура базы данных

`tests`  
Тесты. Содержит код, название и описание теста.

`scales`  
Шкалы теста. Например: `total`, `logic`, `creative`.

`questions`  
Вопросы теста с порядковым номером.

`answer_options`  
Варианты ответов. Каждый вариант имеет код, например `A`, `B`, `C`.

`scoring_rules`  
Правила начисления баллов. Связывают вопрос, код ответа, шкалу и количество
баллов.

`interpretation_rules`  
Правила интерпретации результата по диапазонам баллов.

`test_sessions`  
Сессии прохождения тестов пользователями.

`user_answers`  
Выбранные пользователем коды ответов.

`user_results`  
Итоговые результаты прохождения теста.

## Логика прохождения теста

Пользователь выбирает тест и вводит имя.

Для прохождения создается запись в `test_sessions`.

При ответе на вопрос в `user_answers` сохраняется только код варианта ответа.

Баллы не передаются с клиента. Сервер рассчитывает результат по таблице
`scoring_rules`.

После последнего вопроса создается запись в `user_results`.

## Формат Excel-файла

Excel-файл должен содержать листы:

- `test`
- `scales`
- `questions`
- `answers`
- `scoring`
- `interpretations`

### Лист `test`

| code | title | description |
| --- | --- | --- |
| creator_analyst | Творец или Аналитик? | Определим ваши сильные стороны |

### Лист `scales`

| code | title |
| --- | --- |
| creative | Творческое мышление |
| logic | Аналитическое мышление |

### Лист `questions`

| code | text |
| --- | --- |
| q1 | Что вам ближе? |
| q2 | Как решаете задачи? |

### Лист `answers`

| question_code | answer_code | text |
| --- | --- | --- |
| q1 | A | Рисование |
| q1 | B | Математика |

### Лист `scoring`

| question_code | answer_code | scale_code | points |
| --- | --- | --- | --- |
| q1 | A | creative | 2 |
| q1 | B | logic | 2 |

### Лист `interpretations`

| scale_code | min_score | max_score | text |
| --- | --- | --- | --- |
| creative | 4 | 5 | Вам ближе творческий подход. |
| logic | 4 | 5 | Вам ближе аналитический подход. |
| total | 0 | 100 | Посмотрите на баланс ваших навыков ниже. |

## Импорт Excel

```bash
python import_excel.py path/to/test.xlsx
```
